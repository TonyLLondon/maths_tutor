### What this is about

**Systematic listing** means writing **every** possibility in a neat order so you do not miss any or write the same one twice. It is like making a complete menu of choices.

When one choice follows another (shirt **then** shorts), you often **multiply** the number of options: 2 shirts × 2 shorts = 4 outfits. When you list, use a table or tree: fix the first choice, then run through all second choices.

For codes, digits, or letters, decide rules first: can digits repeat? must all be different? Then list in order (e.g. smallest number first).

This links to counting **[outcomes](../glossary/outcome.md)** in probability — the **[sample space](../glossary/sample-space.md)** is the full list of what can happen.

### Words to know

- **Systematic** — done in a fixed, organised way.
- **Outcome** — one possible result (see [outcome](../glossary/outcome.md)).
- **Sample space** — list of all possible outcomes (see [sample space](../glossary/sample-space.md)).
- **Combination (choice)** — picking from options that go together.
- **Permutation (arrangement)** — order matters (ABC is different from BAC).
- **[Factor](../glossary/factor.md)** — listing all factors is a systematic list too.

### Worked example 1

Archer has a red or blue shirt and black or white shorts. List all outfits.

Fix red: red–black, red–white. Fix blue: blue–black, blue–white.

**Four outfits:** RB, RW, BB, BW (using initials).

### Worked example 2

List all two-digit numbers using digits 2 and 7 once each.

Tens digit 2 → 27. Tens digit 7 → 72.

**Answer:** 27 and 72. (Only two — each digit used once.)

For a coin (H/T) and a die (1–6), list H1, H2, … H6, T1, … T6 — **12 outcomes** in total.

### Tips for practice

- Use columns: first choice down the side, second across the top.
- For “how many ways”, listing first often shows the multiply pattern.
- Check: did any row repeat? did you skip a branch?
- Say the rule aloud: “each digit once” or “digits can repeat”.
- For race orders (ABC, ACB, …), fix who finishes first, then list the rest — same as listing permutations.
- After listing, count entries and compare to a quick multiply check if the question asks “how many”.
